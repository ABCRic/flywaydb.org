How to Contribute

This guide is primarily aimed at people wishing to contribute code to Flyway. If you want to report an issue instead have a look at the Issue Tracker.

Set up your Dev Environment

Dev Environment Setup
Tip: If your feature/bug fix doesn't impact the DB-specific code and you do not wish to set up all the databases, Flyway can be built using mvn clean install -P-MediumTests. This will disable all tests against the DBs.


Your feature/patch/fix/extension

Congratulations ! You now have a fully functional development environment for Flyway. You can now start contributing !


Raise an issue

The first step is to raise an issue in the Issue Tracker. This issue will be the place where you will be able to communicate with Flyway's committers and other users. You can use it to present your ideas and gather feedback.


Clone the repository

The next step is to create a server-side clone the default repository. Log in with your Google account and press the Create Clone button on the Source Tab. Fill in the details and follow the instructions from there.

If you have any problem with Google Code and Mercurial, don't forget to have a look at the Google Code Mercurial FAQ.


Write awesome code

Now is the time to implement your stuff. Commit your changes to your clone and don't forget to pull and merge the changes from the main repo as they happen.

We use IntelliJ to develop Flyway. The free community edition fits the bill perfectly.

All source code is formatted using the default IntelliJ settings.


Publish your changes

Satisfied with your work? Great! Now is the time to show the world! Post a link to your repo in the issue you created, and briefly describe your changes. We'll have a look at them and if they're fine we'll merge them in the mainline and include them in the next release!


Happy coding !
