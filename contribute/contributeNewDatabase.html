Adding support for a new Database

These are the steps to follow to add support for a new database to Flyway.

Prerequisites

  • Enhancement Issue created in Issue Tracker to explain your intent and centralize the discussion (make sure to mention the exact DBMS you which to support, which versions of the DBMS will be supported, if there is a free edition, where can it be downloaded)
  • Dev Environment Setup
  • Flyway building successfully locally
  • Mercurial Clone Repository created on Google Code

Code Changes

  • Extend DbSupport
  • Add new detection to DbSupportFactory
  • Add new createMetaDataTable.sql
  • Extend JdbcTemplate for DB-specific null support
  • Extend SqlStatementBuilder for DB-specific parsing
  • Extend MigrationTestCase and make sure all tests are green
  • Extend ConcurrentMigrationTestCase and make sure test is green

Finishing Touches

  • Make sure your Mercurial repo includes all the latest changes from mainline
  • Push your changes to your mercurial repo
  • Link to your repo from your issue
  • Ask for feedback and rework as necessary

Once this is complete, and the quality meets our standards (in everyone's best interest we can be very picky), it will then become a candidate for inclusion in the next release.
