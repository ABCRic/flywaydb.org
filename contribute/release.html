Releasing Flyway (Committers only)

Prequesites

  1. GPG installed, available on the PATH and a key generated (see https://docs.sonatype.org/display/Repository/Sonatype+OSS+Maven+Repository+Usage+Guide#SonatypeOSSMavenRepositoryUsageGuide-5.Prerequisites)
  2. A checkout of Flyway in two locations: /flyway and /flyway-release
  3. Maven settings.xml contains
    • credentials for Sonatype OSS Repo Hosting
    • credentials for Google Code
    • credentials for Google App Engine
    • the path to the local flyway-release repo checkout
    • the GPG passphrase key:
      • <settings>
    <servers>
        <server>
            <id>sonatype-nexus-snapshots</id>
            <username>myuser</username>
            <password>XXX</password>
        </server>
        <server>
            <id>sonatype-nexus-staging</id>
            <username>myuser</username>
            <password>XXX</password>
        </server>
        <server>
            <id>googlecode</id>
            <username>myuser</username>
            <password>XXX</password>
        </server>
        <server>
            <id>flyway-sample-appengine</id>
            <username>myemail@address.com</username>
            <password>XXX</password>
        </server>
    </servers>
    <profiles>
        <profile>
            <id>flyway-release</id>
            <properties>
                <flyway.releaserepo>/path/to/flyway-release/repo</flyway.releaserepo>
                <gpg.passphrase>XXX</gpg.passphrase>
            </properties>
        </profile>
    </profiles>
</settings>

Required Steps

  1. First, we make sure /flyway is up to date by running hg pull -u
  2. Now under /flyway, we can start the release with mvn release:prepare release:perform -Pflyway-release
  3. We now have to provide the version number for the new release, say 1.3.1 for example
  4. We can use the defaults for the release tag and the new development version
  5. Then, we the release is done, we can push the changes to the POMs by running hg push in /flyway
  6. The old downloads must now be deprecated. We go to http://code.google.com/p/flyway/downloads/list.
  7. We mark each download for the old version as deprecated by removing the Featured tag and adding the Deprecated tag
  8. Now the artifacts, must be promoted to Maven Central by logging in to https://oss.sonatype.org/
  9. We select Staging Repositories on the left and tick the checkbox in front of the open com.googlecode.flyway repository
  10. We then close it (without message) and release it (without message)
  11. Then front page of the wiki must be now be updated
  12. We put the new version number and the date of today in the latest version section
  13. We also update the version number in the Flyway, Flyway Maven Plugin and the Flyway Ant Tasks snippets of the Installation part
  14. We also update the Release Notes by changing the Unreleased Version in the new version with the date of today
  15. The newly released version should also be removed from the Roadmap
  16. Any remaining documentation changes to reflect changes in the code, must now be made to the Wiki
  17. Great, we're done ! We can now announce it on Twitter!
