Ant Tasks

The Flyway Ant Tasks are provided as an AntLib. It is available from the downloads page and from Maven central using this Ivy Configuration:

<dependency org="com.googlecode.flyway" name="flyway-ant" rev="{{ site.flywayVersion }}"/>


Usage

<project ... xmlns:flyway="antlib:com.googlecode.flyway.ant">
   <path id="flyway.lib.path">
       <!-- include all jars from the Flyway AntLib distribution -->
   </path>

   <path id="flyway.classpath">
       <!-- include all jars and directories containing jdbc drivers and migrations -->
   </path>

   <taskdef uri="antlib:com.googlecode.flyway.ant"
            resource="com/googlecode/flyway/ant/antlib.xml"
            classpathref="flyway.lib.path"/>

</project>


Tasks

Name Description
clean Drops all objects in the schema without dropping the schema itself
init Creates and initializes the metadata table in the schema
migrate Migrates the schema to the latest version
validate Validates the applied migrations against the ones available on the classpath
status Prints the current version of the schema
history Prints the full migration history of the schema


Configuration

All tasks can be configured directly using attributes 

<flyway:migrate driver="com.myvendor.Driver" url="jdbc:mydbms:myurl" user="myUser" password="mySecretPwd" />

This configuration can be overridden or supplemented by Ant properties and system properties 

<property name="flyway.user" value="myUser"/>
   <property name="flyway.password" value="mySecretPwd"/>

   <flyway:migrate driver="com.myvendor.Driver" url="jdbc:mydbms:myurl" />
